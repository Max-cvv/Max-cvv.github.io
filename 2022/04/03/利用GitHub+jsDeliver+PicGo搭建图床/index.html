利用GitHub+jsDeliver+PicGo搭建图床

前言

最近突然开始想把自己折腾的东西都记录一下，之前虽然搞过很多东西，但现在让我在搞一遍我还是要去各种搜索。因此打算利用Markdown来记录，而要使用markdown图片问题就必须要解决。如果是本地存储还好，直接插入图片就可以，但如果图片的路径发生改变或者搭建博客，这时候就要利用图床来解决。

准备知识

1、什么是图床？

图床就是存储图片的服务器，现在市面上有很多收费的不收费的图床。收费的如七牛云、阿里云等，免费的大都不稳定。而GitHub作为一个稳定的代码托管平台，那么理论上也可以利用他存储我们的图片，免费而且稳定，唯一的问题就是图片会被公开，所有人都可以看，但既然是要写博客本来就是给别人看的，而且也不会有人闲着看吧。

2、什么是jsDeliver？为什么使用？

jsDeliver为开发者提供免费公共 CDN 加速服务（什么是CDN），他有个很好的特性就是不受墙的影响，因此可以用他链接国外的某些资源，同时恰好他提供了针对GitHub的服务，因此在网站或者博客中使用GitHub中的图片时使用jsDeliver可以提高加载速度。

操作步骤

1、新建GitHub仓库

注册/登录GitHub，新建仓库。

设置仓库的名称等属性。

2、生成token

生成token为PicGo提供访问权限，具体在设置中选择【Developer Settings】

选择【Personal access tokens】，设置属性，选中repo

将生成tokens复制备用

3、配置PicGo

下载PicGo，安装后进行配置。此时自定义域名要按图中配置，这样才能使用jsDeliver的CDN的服务->https://cdn.jsdelivr.net/gh/+仓库名，其中仓库名为GitHub用户名/图床仓库名。

4、Typora+PicGO自动上传

Typora是一款markdown编写软件，虽然更新之后开始收费了，但是可以用之前免费的版本。通过配置Typora可以实现当插入图片时利用PicGo将图片自动上传到Github仓库，首先打开Typora的偏好设置，具体设置如下图：
